安全与防护说明

前端 HTML 只能做基础提示，真正防 DDoS、防 hack 必须靠服务器和 Cloudflare 等服务。

重要：这个文件可以放 GitHub Pages 运行，但它只是前端演示。真正商业网店不要把密码、余额、商品号码、订单验证全部放在前端。

这个页面已经做的基础防护

✅ 输入内容会做简单过滤，避免直接插入危险 HTML。
✅ 登录失败有简单冷却，减少本机暴力尝试。
✅ 订单号码不会写死在页面商品区，购买时才随机生成。
✅ 重要页面有提示：这是虚拟演示，不接真实支付。

真正上线建议

🛡️ 使用 Cloudflare：开启 DDoS Protection、WAF、Bot Fight Mode。
🔐 登录必须放后端：密码要 bcrypt/argon2 加密，不能存在 localStorage。
🧾 商品号码必须放数据库：购买成功后由后端发码，前端不能提前知道全部号码。
🚫 后端做限流：例如每个 IP 每分钟最多请求多少次。
📦 定期备份数据库，开启 2FA，保护 GitHub 和 Cloudflare 账户。
